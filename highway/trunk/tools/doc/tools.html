Highway Build Tools

highway-tools-x.y.z.jar contains 2 distinct tools. These tools are ANT code generation tasks:

  1. ejbgen generates entirely the classes and the EJB descriptors required for accessing the services
  2. vogen generates the value objects

Installation

highwway-tools-x.y.z.jar had no dependencies; it contains all the frameworks it needs. This JAR operates with ANT version 1.6.5 and jdk 1.5. It must be located in the ANT classpath. If you use ANT for the commands, you only have to put it in the ant/lib directory. If you use ANT in Eclipse, you must modify the preferences and add this JAR to the ANT Home Entries. The JAR contains an anttasks.properties file defining the tasks in the org.highway package. To define the tasks simply, all you have to do is put the following line in its build.xml file:

 

<taskdef resource="org/highway/anttasks.properties"/>

ejbgen

The ANT ejbgen file generates the classes and the EJB descriptors. An EJB is generated for each Java interface annotated by org.highway.service.ejb.GenerateEjb and directly or indirectly extending the org.highway.service.Service Java interface. The generation of an EJB for the mypackage.MyServices interface involves the generation of the following files:

  • mypackage/ejb/MyServicesEjbHome.java
  • mypackage/ejb/MyServicesEjbRemote.java
  • mypackage/ejb/MyServicesEjbBean.java
  • mypackage/ejb/MyServicesEjbProxy.java

 

To generate Zip EJB, set the "useCompression" value of the org.highway.service.ejb.GenerateEjb annotation to true. The generation files are the same

 

The following description files are generated for all the services:

  • ejb-jar.xml
  • jboss.xml

 

The ejbgen task takes two parameters:

 

Parameters

Compulsory

Description

inputDir

Yes

Directory of Java sources to be parsed to find the interfaces requiring EJB generation

outputDir

Yes

Destination directory for the Java files generated

 

E.g.:

 

  <ejbgen
    inputDir="src"
    outputDir="bin/gensrc"/>

vogen

The ANT vogen task generates your application’s value objects.

 

The definition and therefore the generation of a ValueObject type of class is done via a Java definition interface. This interface has a name ending in Def and extends directly or indirectly the org.highway.bean.ValueObject interface. For example, the mypackage.MyValueObjectDef package defines the mypackage.MyValueObject class.

 

The ValueObject type of class generated by vogen will implement the Def interface it defines. For example, the mypackage.MyValueObject class will implement the mypackage.MyValueObjectDef interface. NB: the class generated implements all the interfaces the Def interface inherits at the same time.

 

To define the value object properties: you must define its getters. The interface JavaBean getter type of method will define the value object properties. For example:

 

  String getFirstName(); // defines the JavaBean property firstName

 

There are two ways of having a value object generated by vogen inherited. You can use the highway.vogen.superclass annotation and therefore give the name of the class to be extended. You can also use inheritance in the definition interfaces. If ADef extends BDef, the A class will extend the B class. If both methods are used the tag has priority.

If the developer does not specify the super class, the class generated extends org.highway.bean.ValueObjectAbstract.

 

You can specify the value object characteristics and its properties with the annotations defined in the package org.highway.bean. You can also specify mapping information for a value object and its properties. For futher information, read the javadoc in the annotation of the package org.highway.database.

 

vogen generates the following files for the mypackage.MyValueObjectDef interface:

 

File

Description

When

mypackage/MyValueObject.java

Implementation of this value object

Default behavior

mypackage/MyValueObjectBase.java

Base implementation of this value object

If annotated with org.highway.vogen.GenerateBaseOnly

mypackage/MyValueObject.hbm.xml

Contains the Hibernate mapping for this value object

If the definition interface is annotated with org.highway.database.Mapped

 

The definition interface contains all the information required for generating these files as Java constants and annotations. For more information on the coding for a definition interface, see the javadoc and the specification of each annotation.

 

An XML Hibernate configuration file is generated for all the definition interfaces. This file is generated by default in the root of the output directory (outputDir) with the name of “hibernate.cfg.xml”. It contains the list of Hibernate mapping files generated for each definition interface.

 

The vogen task takes two parameters:

 

Parameters

Compulsory

Description

inputDir

Yes

Directory of Java sources to be parsed to find the value object definition interfaces

outputDir

Yes

Destination directory for the Java files generated

 

E.g.:

 

  <vogen
    inputDir="src"
    outputDir="bin/gensrc"/>
