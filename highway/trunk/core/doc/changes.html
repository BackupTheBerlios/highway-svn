CORE Revision History

Version 1.4.7

Defects fixed

  • Unregistered defect : JavadocMetadataManager - Performance problem in getPropertyMetaValue(Class, String, String) method.
  • Unregistered defect : Log4jDebugLog - Bug in getLog4jLogger(Class) method, used by DebugInterceptor.

New features

  • Add new package : org.highway.eai - provides EAI integration.
  • SuperviseHome provides shortcut methods which delegate to the default Supervisor.

Dependencies

  • CORE 1.4.7 must be used in conjunction with TOOLS 1.4.5.

Version 1.4.6

Defects fixed

  • Defect #105 : DateHelper methods are now in English.
  • Defect #126 : problem while validating a bunch of value objects.

New features

  • Evo #86 : value object string properties may be trimmed through an Hibernate type (TrimmedStringHibernateType) or an Hibernate interceptor (TrimInterceptor).
  • Creation of trim methods in StringHelper.

Dependencies

  • CORE 1.4.6 must be used in conjunction with TOOLS 1.4.5.
  • Several libraries CORE relies on were upgraded : commons-collections, commons-beanutils, commons-lang and ognl.

Version 1.4.5

Defects fixed

  • Defect 123 : NullPointerException fired when using EJB compression with service without parameter.

New features

  • Add new Package : org.highway.cache
  • Add DataCache : First implementation of dataCache interface.

Dependencies

  • CORE 1.4.5 must be used in conjunction with TOOLS 1.4.4.

Version 1.4.4

Defects fixed

  • Unregistered defect : fixed Blob and Clob Hibernate property mapping.
  • Unregistered defect : the jar is now compiled in debug mode.

Dependencies

  • CORE 1.4.4 must be used in conjunction with TOOLS 1.4.3.

Version 1.4.3

Defects fixed

  • Defect #58 : "" and null are now managed in a different way by value objects.

New features

  • Evo #88 : Hibernate conversion classes for blob and clob.

Dependencies

  • CORE 1.4.3 must be used in conjunction with TOOLS 1.4.3.

Version 1.4.2

Defects fixed

New features

  • Add EJB compression : An implementation for ejb compression is now available. To use it, replace the tag highway.service.generate.ejb by the tag highway.service.generate.ejbzip. 3 new classes are added : ZipEjbProxy and ZipEjbBean they are the abstract classes for ejb compressed service access, ZipEjbProxyRequest this class is used to serialize and zip the parameters and to call the service zip modified internal method.
  • io : New package, it is now the reference package for all the application's io. Some classes of this package replace classes from helper package. A standard implementation of Serializer and Deserializer output can be found in this package.

Dependencies

  • CORE 1.4.2 must be used in conjunction with TOOLS 1.4.2.

Version 1.4.1

Defects fixed

  • Defect #72 : Log4jSupervisor throws a NullPointerException.
  • Defect #73 : problem while openning a connection on XA Mail if the previous transaction failed.
  • MapOfSetImpl now implements MapOfSet.

New features

  • Debug : DebugHome now contains all the short cut methods to the default DebugLog instance.
  • Validation : ValidateContext and ValidateProblem are now Serializable and can be passed from the client to the server and vice-versa.
  • Validation : you can reset a ValidateContext object to avoid to many object creation. One reset method to totally reset the object, a second reset method to only reset the validation reset but keep the validation parameters.
  • Validation : you can add validators directly on the ValidateContext object to better customize the validation of objects depending on the validation context.
  • Helpers : the context object used in ResourceHelper methods can now be a String path, a Class object, a package object or any kind of object.
  • XA Mail : the deployment descriptor file is now compatible with WSAD.

Dependencies

  • None.
